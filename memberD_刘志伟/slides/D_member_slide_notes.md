# D成员 PPT 页素材

## 第9页：评价指标和实验矩阵

标题：评价指标：同时衡量安全性与可用性

建议放两块内容：

1. 指标表：ASR、LRate、Utility、ORR、Score。
2. 实验矩阵：D0 无防御，D1 Prompt，D2 输入过滤，D3 输出检查，D4 组合防御。

讲稿：我们把攻击样本和正常样本分开评测。攻击样本主要看 ASR 和 LRate，正常样本主要看 Utility 和 ORR。这样可以避免只追求安全导致系统把正常问题也拒掉。

## 第10页：结果图表

标题：D0-D4 对比结果

建议放四张图或两张组合图：

- ASR 对比。
- LRate 对比。
- Utility 与 ORR 对比。
- Score 对比。

讲稿：理想结果是 D4 的 ASR 和 LRate 明显低于 D0，同时 Utility 不大幅下降。如果 ORR 上升，需要说明组合防御的代价。

## 第11页：结论、局限和伦理

结论模板：

在当前中文课程问答 RAG 样本上，单一防御可以降低部分风险，但组合防御能同时覆盖提示词、检索文档和最终输出三个环节。局限是样本规模有限，自动评测仍需人工复核，后续可以加入 LLM 分类器和更丰富的混淆样本。

伦理说明：

所有密钥、系统提示词和攻击目标均为模拟内容；实验仅在本地课程系统中进行，不测试真实线上系统，不收集真实个人隐私。
